# From Zero to Expert Technical Writer
A structured portfolio and learning hub documenting the journey to master technical writing.

This repository powers the content for my personal technical writing website.

## 🎯 About This Project
Aligned with my website's information architecture, this repo serves as the **single source of truth (SSOT)** for my learning progress and professional portfolio. It tracks my evolution from a beginner to an expert-level technical writer by building four core competencies:

1. **Structured Learning** (Systematic path building)
2. **Reusable Templates** (Standardized writing assets)
3. **Hands-on Projects** (Real-world documentation samples)
4. **Skill Mastery** (Expert-level competency checklist)

## 📂 Repository Structure (Map to Website IA)
The folder structure below directly mirrors the main sections of the website for easy navigation.

| Directory | Corresponding Website Section | Description |
| :--- | :--- | :--- |
| `/learning-path` | **Learning Path** | Step-by-step notes, course materials, and progress trackers for core technical writing skills. |
| `/writing-templates` | **Templates Library** | Production-ready templates (API docs, user guides, release notes) following industry style guides. |
| `/practice-projects` | **Project Portfolio** | Complete documentation projects, including source files, revision history, and publishing demos. |
| `/skill-checklists` | **Mastery Checklist** | Self-assessment rubrics to measure progress toward expert-level proficiency. |

## 📈 Growth Milestones
Tracks the key phases of my journey as defined in the website's learning path:
- [ ] **Phase 1: Foundations** - Master audience analysis, content structure, and Markdown/Git fundamentals.
- [ ] **Phase 2: Tools & Workflow** - Proficient in documentation systems (e.g., MkDocs/Docusaurus) and CI/CD for docs.
- [ ] **Phase 3: Specialization** - Deep dive into [Your Focus: e.g., REST API Documentation / Cloud Services].
- [ ] **Phase 4: Expertise** - Master advanced topics (localization, accessibility, content strategy).

## 🛠️ Technical Stack

- **Version Control**: Git, GitHub
- **Documentation as Code (DaC)**: [MkDocs / Sphinx / Docusaurus]
